При добавлении шрифтов для TEXT-элементов раздаток (RPGLab, [[konva-transform-no-bubble]]) в HandoutCanvas/TextProperties наткнулся на две неочевидные вещи.

**1. Название/жанр шрифта не гарантирует кириллицу.** Пользователь попросил Special Elite, Courier Prime и Kalam (все — популярные тематические шрифты типа «печатная машинка»/«рукописный»). У всех трёх в Google Fonts **нет кириллических глифов вообще** — только `latin`/`latin-ext` (Special Elite, Courier Prime) или `devanagari`/`latin` (Kalam). Проверяется по составу `@font-face` в `https://fonts.googleapis.com/css2?family=...` — ищи блоки `/* cyrillic */`. Заменил на PT Mono/Cousine/Overpass Mono (печатная машинка) и Caveat/Marck Script/Neucha (рукописный) — у всех есть настоящий cyrillic-блок.

**Why:** легко предположить, что раз шрифт «под старую печатную машинку», значит он от полнофункциональной шрифтовой системы с кириллицей — не так, многие англоязычные декоративные шрифты этим не озаботились.
**How to apply:** перед тем как обещать пользователю конкретный шрифт с поддержкой кириллицы — сначала скачать/проверить его CSS, а не полагаться на название.

**2. Canvas (Konva) не ждёт загрузку кастомного `@font-face` шрифта.** В отличие от `<img>`/обычного DOM-текста, `CanvasRenderingContext2D.fillText` рисует ТЕКУЩИМ (уже загруженным/фолбэк) шрифтом синхронно — если файл шрифта ещё не догружен браузером, canvas не ждёт и не перерисовывается сам, когда шрифт наконец готов.

Два места, где это стреляет:
- Живой холст: элемент с только что выбранным кастомным шрифтом рисуется фолбэком, пока шрифт не догрузится — нужна подписка на `document.fonts.addEventListener('loadingdone', ...)` → forceDraw слоя.
- Экспорт (`stage.toDataURL()`): если пользователь выбрал шрифт и сразу жмёт «Экспорт», файл может быть ещё не запрошен браузером вообще — `document.fonts.ready` ждёт только уже начатые загрузки, поэтому нужно ЯВНО `document.fonts.load('16px "FontName"')` для каждого использованного шрифта, потом `await document.fonts.ready`, и только затем снимать canvas.

**Why:** результат бага — тихий (без ошибки) рендер неправильным шрифтом, легко пропустить при беглой проверке.
**How to apply:** в любом Canvas/Konva-проекте с кастомными веб-шрифтами — всегда явно грузить шрифты перед первой отрисовкой текста и перед любым экспортом снимка канваса.
